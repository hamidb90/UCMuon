# Source-spectrum validation

Compares the muon spectrum parametrizations implemented in the UCMuon
generator (`src/generator/ucmuon_source_module.f90`, spectrum modes of
`build_cosmoaleph_cdf`) against the original publications and the
underlying sea-level measurements. Complements the transport benchmark in
`benchmark/codes/` — here it is the *source* term that is validated.

## Files

| File | Purpose |
|------|---------|
| `ucmuon_spectra.py` | Python ports of the Fortran spectrum modes (1, 2, 4–7), the PARMA Mode 3 loader, the CosmoALEPH charge-ratio table, and the Bugaev 1998 reference fit |
| `make_parma_spectrum.py` | Compiles + runs the real PARMA Fortran (`src/parma/`) to cache the Mode 3 vertical spectrum and angular factors (needs `gfortran`; run once, commit the CSVs) |
| `parma_angular_spec.f90` | Small Fortran helper: PARMA muon angular factor `getSpecAngFinal` vs zenith angle, used by `make_parma_spectrum.py` |
| `extract_reyna_fig3.py` | Digitizes Fig. 3 of Reyna 2006 from the **vector** PDF (`references/source/Reyna-B.pdf`) — marker centres are exact path coordinates, not pixel estimates |
| `plot_source_spectra.py` | Produces the comparison figures below |
| `data/parma_vertical_spectrum.csv` | PARMA/EXPACS (Sato 2015) vertical muon spectrum, generated by `make_parma_spectrum.py` at the sea-level reference site |
| `data/parma_angular.csv` | PARMA/EXPACS muon angular factor F_ang(θ) at p = 1/10/100 GeV/c (same site), for the zenith figure |
| `data/cosmoaleph2013_table1.csv` | CosmoALEPH spectrum + charge ratio, transcribed verbatim from Schmelling et al. 2013, Table 1 |
| `data/reyna2006_fig3_data.csv` | 208 digitized points from 11 experiments (Nandi & Sinha, MARS, OKAYAMA 0–80°, Kellogg 30/75°, Kiel–DESY, MUTRON), scaled to vertical with cos³θ as published |
| `data/reyna2006_fig3_bestfit.csv` | Reyna's best-fit curve as drawn in the paper |

Run `python3 extract_reyna_fig3.py` (needs `pymupdf`) to regenerate the
digitized CSVs, then `python3 plot_source_spectra.py` for the figures.

## Digitization accuracy

Axis calibration uses the figure's major tick marks recovered from the PDF
drawing commands. Validation: the digitized "Best Fit" curve reproduces
Reyna's published Eq. 6–7 with median ratio **1.001 ± 0.006** over
ζ = 0.3–1500 GeV, so digitization errors are negligible compared with the
experimental error bars. `figures/qa_reyna_fig3_overlay.png` overlays every
extracted point on the original figure for visual QA.

## Figures (`figures/`)

- **fig_source_spectrum** — vertical differential intensity, all modes vs.
  all data, with a ratio-to-Mode-7 panel. Shows: Guan/Frosin/Reyna agree
  with data at the ±10 % level over 1–2000 GeV/c; plain Gaisser (Mode 6)
  overshoots below ~30 GeV (no low-energy correction — by construction);
  the Mode 1 CosmoALEPH power law is only valid above ~100 GeV/c.
- **fig_source_spectrum_p3** — p³-weighted version; highlights the
  high-momentum region covered by CosmoALEPH (which sits ~5 % above the
  parametrizations, consistent with the paper's own comparison).
- **fig_charge_ratio** — the generator's `charge_ratio_from_p` step table
  against the CosmoALEPH measurement it was built from.
- **fig_zenith_dependence** — zenith dependence of the physics models
  at 1/10/100 GeV/c (Guan cos θ\* convention vs. Reyna cos³θ scaling), now
  including PARMA (Mode 3) from `getSpecAngFinal` cached in
  `data/parma_angular.csv`. Mode 6 stands apart by construction: the plain
  Gaisser formula contains only the sec θ production enhancement and neither
  muon decay nor atmospheric energy loss, so it stays flat at 1 GeV and rises
  unchecked at large angles instead of turning over like the corrected
  models. PARMA shows a small cusp at θ = 60° (cos θ = 0.5), a breakpoint in
  its piecewise angular fit; the overall rise above unity at 100 GeV/c is the
  physical sec θ enhancement, shared with Guan and Reyna.
- **fig_mode\<N\>_\<name\>** — one figure per spectrum mode (1, 2, 3, 4–7):
  the single model against all data with a data/model ratio panel
  (±10 % band marked, gray shading outside the model's validity range).
  Mode 3 (PARMA) tracks the data within ±20 % from ~0.3 GeV/c out to
  ~1 TeV/c — the only model that also follows the low-momentum turnover —
  then underpredicts the high-energy tail.

## Notes

- Mode 2 (E⁻³·⁷ power law) is a sampling shape with no absolute
  normalization (Kudryavtsev/MUSIC convention); it is drawn pinned to
  Mode 7 at 100 GeV/c.
- Mode 3 (PARMA/EXPACS) has no closed form — it is the numerical PARMA
  model (`src/parma/`). Rather than re-implement it in Python, the real
  Fortran is run once by `make_parma_spectrum.py` and cached to CSV, which
  `ucmuon_spectra.mode3_parma` then loads like any other reference curve.
- Mode 8 (cosmic electrons) is not a muon spectrum and is out of scope here.
- Reyna's Fig. 3 collapses all zenith angles onto the vertical via
  I_V = I/cos³θ at ζ = p·cosθ, so the digitized points are directly
  comparable to the vertical-spectrum modes.
