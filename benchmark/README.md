# UCMuon transport-engine benchmark

Six-code comparison of muon transport through Standard Rock — the validation
backbone of the UCMuon paper. It cross-checks the four UCMuon engines against
two independent external Monte Carlo references.

| Code | Role | `codes/` folder |
|------|------|-----------------|
| **Geant4** 11.2 | external reference | `codes/geant4/` |
| **PHITS** 3.36 | external reference | `codes/phits/` |
| **MUSIC** (Engine 1) | UCMuon | `codes/music/` |
| **PROPOSAL** (Engine 3) | UCMuon | `codes/proposal/` |
| **Bethe–Bloch** (Engine 2) | UCMuon | `codes/bethe_bloch/` |
| **UCMuon-MC** (Engine 1, flagship) | UCMuon | `codes/ucmuon_stochastic/` |

**Population:** six monoenergetic vertical beams (5, 10, 20, 50, 100, 300 GeV),
10⁵ muons each → N = 6×10⁵; Standard Rock (ρ = 2.65 g cm⁻³); scored at 1, 10,
25, 50, 100, 200 m. Every code receives the **identical** source population
(`benchmark_surface.dat`, converted per code).

> **Authoritative run: v2 (2026-05-25)** — `reports/BENCHMARK_FEEDBACK.md`.
> Its 100 m (265 MWE) transmissions (G4 32.7 %, PHITS 32.6 %, MUSIC 32.9 %,
> PROPOSAL 32.8 %, BB 33.3 %, UCMuon 32.3 %) are Table 1 of the manuscript.
> Raw `.dat` files live in the git-ignored scratch directory
> `benchmark/geant4_muon_rock_v5/` (≈8 GB, never tracked).

## Folder map

```
benchmark/
├── README.md          ← this file (overview + roadmap)
├── codes/             ← one subfolder per transport code, each with its own README
│   ├── geant4/          MuonRock.cc, src/, include/, CMakeLists.txt, macros
│   ├── phits/           muon_rock.inp, source converter, tally parser
│   ├── music/           how to run Engine 1 (code lives in main repo)
│   ├── proposal/        how to run Engine 3
│   ├── bethe_bloch/     how to run Engine 2
│   └── ucmuon_stochastic/  how to run Engine 4
├── analysis/          ← cross-code scripts (benchmark_analysis.py, run_benchmark.sh, …)
├── sources/           ← source-spectrum validation: UCMuon spectrum modes vs.
│                        original data (CosmoALEPH table, digitized Reyna 2006
│                        compilation); see sources/README.md
├── results/           ← summary CSVs + per-code timing files
├── figures/           ← v2_six_code/ (canonical) and four_code/ comparison plots
└── reports/           ← write-ups; BENCHMARK_FEEDBACK.md is the v2 reference
```

> The ~8 GB of raw per-event `*_bench_<depth>m.dat` is **not** tracked in git.
> It lives in the git-ignored scratch tree `benchmark/geant4_muon_rock_v5/`,
> regenerated by the per-code steps. This folder holds the distilled products.

## Roadmap — reproducing the benchmark end to end

1. **Common source.** Generate `benchmark_surface.dat` (6 monoenergetic beams,
   600 k muons) with the UCMuon generator, then convert it per code
   (`codes/phits/convert_source_to_phits.py`; the Geant4 converter in the
   scratch tree).
2. **Run each code** using its `codes/<engine>/README.md`:
   - external: `codes/geant4/`, `codes/phits/`
   - UCMuon engines: one `tools/compare_engines.py` call per engine (`music`,
     `proposal`, `bb`, `ucmuon`).
3. **Analyse.** `cd analysis && bash run_benchmark.sh` → comparison figures
   (`figures/`) + `benchmark_summary.csv`.
4. **Paper tables.** `python3 manuscript/scripts/make_survival_table.py` regenerates the
   depth-resolved survival table (reads from `benchmark/geant4_muon_rock_v5/`);
   the exit-KE table is in `manuscript/tab_exitke.tex`.
5. **Open items** before final submission (also in `manuscript/PUBLICATION_TODO.md`):
   PROPOSAL X₀ MCS fix; PHITS −12 % exit-KE single-layer test.
